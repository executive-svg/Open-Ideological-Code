# Правила участия / Contributing Guidelines

## Язык

Рабочие языки проекта: **русский** и **английский**. Базы знаний и обсуждения ведутся на русском. Финальные препринты готовятся на английском.

## Как присоединиться

1. Свяжитесь с координатором проекта (Igor Kaminnyk)
2. Получите приглашение в репозиторий (Collaborator access)
3. Ознакомьтесь с базой знаний по вашему направлению в папке `directions/`

## Рабочий цикл

Каждое направление проходит через цикл из 9-10 сессий (подробнее — в `revision-plan.md`).

### До сессии
- Координатор открывает **Issue** на GitHub с темой сессии и вопросами для обсуждения
- Прочитайте раздел базы знаний, указанный в Issue
- Подготовьте комментарии, возражения и источники
- Если есть материалы — прикрепите их к Issue

### Во время сессии (Zoom)
- Обсуждаем конкретный раздел базы знаний
- Координатор фиксирует ключевые решения
- Определяем, кто вносит правки по итогам

### После сессии
1. Ответственный эксперт создаёт **ветку** (например, `session-3-subjectlessness`)
2. Вносит изменения в `knowledge-base.md`
3. Отправляет **Pull Request** со ссылкой на Issue
4. Координатор проверяет и мержит PR в `main`
5. Координатор обновляет `discussion-log.md` (дата, участники, решения, ссылка на PR)

### Упрощённый вариант (без Git на компьютере)
Если вы не работали с Git раньше — можно редактировать прямо на GitHub:
1. Откройте нужный файл на github.com
2. Нажмите иконку карандаша (Edit)
3. Внесите правки
4. Нажмите **Propose changes** — GitHub автоматически создаст ветку и Pull Request
5. Координатор проверит и примет изменения

## Структура папок

- `directions/<direction>/knowledge-base.md` — база знаний (основной документ)
- `directions/<direction>/revision-plan.md` — план ревизии по сессиям
- `directions/<direction>/discussion-log.md` — лог обсуждений и решений
- `directions/<direction>/drafts/` — черновики статей
- `sessions/` — протоколы Zoom-сессий
- `preprints/` — финальные версии препринтов
- `templates/` — шаблоны протоколов и рецензий

## Правила редактирования

- Все содержательные изменения — через **Pull Request**, не прямым коммитом в `main`
- Не удаляйте чужой текст без обсуждения — комментируйте или предлагайте альтернативу
- Крупные структурные изменения обсуждайте на сессии или в Issue
- Описывайте изменения в commit message на русском или английском

## Авторство

Все участники, внёсшие существенный вклад, указываются как соавторы в публикациях. Порядок авторства определяется по итогам работы над каждой конкретной статьёй.
